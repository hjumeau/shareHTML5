L'objectif de cette soirée sur le thème HTML5, est de vous sensibiliser à ce nouveau standard du Web. Nous allons démystifier ...

va attiser votre curiosité et permettra d'identifier des sujets à approfondir...

Constat

Divergence grandissante entre

  • les possibilités du langage HTML4
  • les applications web actuelles


Résultat : une belle pagaille

  • afflux de technologies côté client
  • le plein de plug-ins (flash, silverlight…)
  • divergences entre navigateurs

Naissance d’HTML5

Confusion autour de HTML 5
Travaux du WHATWG:
WebApps 1.0 et WebForms 2.0
Crée par Apple, Opera et Mozilla
Repris par le W3C comme base de travail

Candidate Recommandation : 2012
deux versions 5.0 et 5.1
déjà implémenté partiellement
chrome, safari, opera, firefox, IE9
test avec http://html5test.com/

Adoption complète (2022) ? HTML 5.0 2014
HTML 5.1 2016

Lignes directrices

Compatibilité avec l’existant
éléments obsolètes (a ne pas utiliser)
mais pas « deprecated » (le navigateur les affiche quand même)

Ne pas réinventer la roue
garder ce qui fonctionnait avant

Aboutir à une spécificateur réelle
préférer le pragmatisme à la pureté

Aller vers plus de simplicité : 

        <script type='text/javascript' src='file.js'/>
        //Devient :
        <script src='file.js'/>
Documenter les API Javascript

HTML 5 : principales fonctionnalités

  • Structure : nouvelles balises
  • Intégration audio et video
  • Images : SVG, Canvas, WebGL
  • Web Forms 2.0
    • nouveaux types (dates, sliders …)
    • validation
    • autofocus
    • autocomplete
  • Microformats

CSS 3 : fonctionnalités

  • Nouveaux sélecteurs
  • Organisation en colonnes
  • Gestion de la typographie
    • ombres, nouvelles fonts …
  • Gestion des couleurs améliorée
    • gradients, saturation
  • Effets visuels :
    • Arrondis
    • Opacité sur différents plans
    • Transitions
    • Transformations
    • Animations

JS API

  • Gestion d’un mode off-line
  • Stockage de données
    • ombres, nouvelles fonts …
  • Communication
    • Web Sockets
  • Notification
  • Tâches de fond(Web workers)
  • Géolocalisation

Conclusion

  • HTML5 est présenté comme le tueur de flash
    • De nombreuses nouveautés (HTML5, CSS3, JS API)
    • Mais de écueils a éviter
      • problèmes d’implémentation et de standardisation
  • Tout ce que nous allons couvrir est susceptible
    • de subir des modifications
    • de ne pas fonctionner dans tous les navigateurs
    • ou de fonctionner différemment
